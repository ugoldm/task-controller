# Task Controller — требования к сервису

Личный сервис для отслеживания задач продакт-менеджера. Заменяет «Избранное» в Telegram.

## 1. Проблема

Сейчас план дня ведётся в «Избранном» Telegram: список с зачёркиванием.

Боли:
- **Потеря перенесённого.** 30–50% задач за день не выполняется. На следующий день они либо теряются, либо вручную перепечатываются.
- **Нет контекста.** К задаче невозможно нормально прикрепить мысли, ссылки, ход обсуждения. А контекст по чужим стримам (матричные подчинённые) и своим стратегическим стримам нужно помнить.
- **Хаотичный захват.** Новые задачи рождаются в рабочих чатах в течение дня; их нужно быстро фиксировать, не теряя формулировку.

## 2. Пользователь и контекст

Один пользователь — продакт-менеджер в крупном цифровом банке. Параллельно ведёт:
- **Чужие стримы** — задачи людей в проектном (матричном) подчинении: погружение, согласования, мнение.
- **Свои стримы** — в основном стратегические, решаются им от начала до конца.

Сервис персональный, мультипользовательность не требуется.

## 3. Ключевые принципы

1. **Ничего не теряется.** Незавершённая задача не исчезает — она переносится явно и осознанно.
2. **Захват за секунды.** Добавить задачу должно быть так же быстро, как написать строчку в Telegram.
3. **Контекст живёт рядом с задачей.** Заметки, мысли и ссылки — часть карточки задачи.
4. **Без лишней церемонии.** Дедлайны, статусы, поля — опциональны. Базовый поток остаётся простым списком.

## 4. Функциональные требования

### 4.1 Структура данных

- **Стрим (проект)** — контейнер для задач. Поля: название, цвет, тип (`свой` / `проектный`).
- **Задача** — поля:
  - `title` — формулировка (обязательно)
  - `stream` — принадлежность к стриму (обязательно)
  - `status` — `открыта` / `выполнена` / `отложена`
  - `notes` — заметки/контекст (свободный текст, дополняется в течение жизни задачи)
  - `inToday` — включена ли в план на сегодня
  - `order` — позиция задачи внутри стрима (приоритет, задаётся вручную)
  - `deadline` — дата (опционально, для редких случаев)
  - `carryCount` — сколько раз переносилась (считается автоматически)
  - `createdAt`, `completedAt`

### 4.2 Вид «Сегодня» (главный экран)

- Список задач, включённых в план на сегодня, **сгруппированный по стримам**.
- Отметка выполнения в один клик (чекбокс / зачёркивание).
- Перенесённые задачи визуально помечены (бейдж «перенесена ×N»).
- Быстрое поле добавления задачи: ввод строки + выбор стрима.
- Внутри стрима задачи можно двигать вверх/вниз перетаскиванием (drag-and-drop), задавая приоритет; работает в обоих видах — «Сегодня» и «Стримы».
- Задачу можно быстро убрать из плана на сегодня (одно действие на карточке задачи и в самой строке) — при этом она остаётся в своём стриме (бэклоге).
- Задачи с дедлайном на сегодня/просроченные — визуально выделены.

### 4.3 Вид «Стримы» (бэклог)

- Все стримы с раскрываемыми списками всех задач (не только сегодняшних).
- Из бэклога задачу можно одним действием добавить в план на сегодня.
- Создание, редактирование и удаление стримов. При удалении непустого стрима — подтверждение.

### 4.4 Карточка задачи

- Открывается панелью (drawer) поверх любого вида.
- Редактирование формулировки, стрима, статуса, дедлайна.
- **Блок заметок/контекста** — основное: мысли по задаче, ссылки, ход обсуждения. Дополняется постепенно.
- История: дата создания, число переносов.

### 4.5 Утреннее ревью (AI)

Запускается утром (вручную или по расписанию):
1. Показывает все незавершённые задачи со вчера (и ранее).
2. По каждой — выбор: `взять на сегодня` / `оставить в бэклоге` / `выполнена` / `удалить`.
3. AI предлагает черновой план дня и подсвечивает риск перегруза, если задач набрано слишком много.
4. Результат — собранный вид «Сегодня».

### 4.6 Добавление задачи

Действие «Новая задача» (кнопка «+» / пункт меню) предлагает два способа:

**a) Ввести вручную** — пользователь сам задаёт формулировку и стрим.

**b) Разобрать из текста (AI)**
- Поле для произвольного текста — не обязательно переписка из чата; это может быть свободно написанное описание того, что нужно сделать.
- AI распознаёт в тексте задачи и **разбивает его на несколько отдельных задач**.
- По каждой распознанной задаче предлагается формулировка и стрим.
- Пользователь правит формулировки, меняет стримы, снимает галочку с лишнего — затем задачи создаются разом.

## 5. Нефункциональные требования

- **Платформа:** адаптивное веб-приложение — десктоп (sidebar-навигация) и мобайл (нижняя навигация, крупные элементы).
- **Скорость:** добавление задачи и отметка выполнения — без перезагрузок, мгновенно.
- **Приватность:** в задачах могут быть рабочие детали банка → данные не должны уходить к неконтролируемым третьим сторонам. Хостинг — см. раздел 7.
- **Офлайн-устойчивость (желательно):** PWA, чтение и базовое редактирование без сети.
- **Не требуется:** аналитика/ретроспектива, мультипользовательность, нативные приложения.

## 6. Варианты архитектуры

| Вариант | Стек | Плюсы | Минусы |
|---|---|---|---|
| **A. Лёгкий self-hosted** (рекомендуется) | React/Svelte SPA + Node (Fastify) API + SQLite/Postgres, Docker на личном VPS | Полный контроль данных, дёшево, PWA из коробки | Нужно поднять и поддерживать VPS |
| **B. Локальный** | Тот же фронт, бэкенд локально, SQLite-файл | Данные не покидают устройство | Нет синхронизации десктоп ↔ мобайл |
| **C. Serverless** | SPA + Cloudflare/Vercel + managed-БД | Минимум эксплуатации | Данные у внешнего провайдера — спорно для банка |

**AI-функции** (утреннее ревью, разбор задачи из свободного текста) во всех вариантах — отдельный модуль через Claude API. Важно: AI-вызовы идут через бэкенд, в промпт уходит только то, что пользователь явно отправил.

## 7. Открытый вопрос — хостинг

Нужно решить до старта разработки. Рекомендация: **Вариант A** (личный VPS, Docker) — даёт синхронизацию десктоп/мобайл и сохраняет контроль над данными. Уточнить, нет ли в банке политики, запрещающей выносить рабочий контекст на личную инфраструктуру; если есть — Вариант B.

## 8. Предлагаемый объём MVP

1. Стримы + задачи + вид «Сегодня» с группировкой и ручным добавлением задачи.
2. Карточка задачи с заметками.
3. Перенос: незавершённые задачи не теряются, бейдж «перенесена ×N».
4. Утреннее ревью (сначала можно без AI — простой мастер разбора вчерашнего).

Затем: AI в утреннем ревью и разбор задачи из свободного текста, PWA/офлайн, дедлайны с уведомлениями.
